Getting started An overview of alv-ch-ng.

Bower

Install and manage alv-ch-ng's Less, CSS, JavaScript, and fonts using Bower. 
$ bower install alv-ch-ng

Grunt

There are several Grunt tasks to improve the quality and the progress developing alv-ch-ng.
"Quality" tasks
  • $ grunt unit-test starts Jasmine test
  • $ grunt jshint-test starts JSHint test
  • $ grunt lesslint-test starts LESS Lint test
  • $ grunt all-test runs all tests
Development tasks
  • $ grunt prepare prepares the demo application, alv-ch-ng.demo
  • $ grunt build build all necessery files (angular, alv-ch-ng, and more) for the demo application
  • $ grunt dev starts the BrowserSync task for developing. Runs the necessary tests from above after changing the sourcecode and updates the browser view.

Compiling LESS

alv-ch-ng can be used in at least two ways: with the compiled CSS or with the source Less files. To compile the Less files, visit the official Less website to run the necessary commands.

Run alv-ch-ng with the Less files


    <link rel="stylesheet/less" type="text/css" href="/src/less/alv.less">
    <script type="text/javascript">
        less = {
            env: "development",
            async: false,
            fileAsync: false,
            poll: 1000,
            functions: {},
            dumpLineNumbers: "comments",
            relativeUrls: false,
            rootpath: ":/a.com/"
        };
    </script>
    <script type="text/javascript" src="/lib/less/dist/less-1.7.0.min.js"></script>
For more information, please visit the official Less websites section: Client-side usage.
